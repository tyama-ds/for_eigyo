# Kaleido Agents — マルチエージェント・オーケストレータ

依頼を投げると、オーケストレータが計画を立て、**サブエージェント**が
**ツールモジュール**を使い分けて依頼を完結させるシステム。
実行の様子（計画 → 各ステップ → ツール入出力 → レビュー）を
万華鏡のような多色 UI でリアルタイムに可視化する。

```bash
python kaleido-agents/server.py          # http://127.0.0.1:8790
python kaleido-agents/server.py --port 9400 --open
```

- **標準ライブラリのみ**（pip install 不要）。127.0.0.1 にのみ bind し外部公開しない
- **LLM なしでも動く**。ルールベースの計画立案とツール実行だけで依頼を完結できる
- ⚙️ 設定から LLM（**ローカルLLM** / OpenAI 互換クラウド / Anthropic Claude）を接続すると、
  計画立案・要約・自由質問・最終回答の文章生成が LLM で強化される
- ローカルLLM の扱いは llmlab / news-portal と同じ流儀:
  OpenAI 互換エンドポイント・APIキー任意・**プロキシ非経由で直結**・
  推論系モデル（DeepSeek-R1 / QwQ 等）の `<think>` 分離に対応

## アーキテクチャ

```
依頼 ──▶ オーケストレータ（実行エンジン, app.js）
           │ 1. 🧭 プランナー   … 依頼を解析し「どのツールを使うか」に分解（ルール / LLM）
           │ 2. オーケストレータが担当エージェントを動的に割り当て
           │      各ツールは担当候補（得意順）を持ち、有効なエージェントの中から
           │      計画内の負荷が最小のものを選ぶ。LLM プランナーの指名は
           │      担当可能なら尊重。割り当て理由は実行ログに表示
           │ 3. 各サブエージェントがツールを実行
           │      🔎 リサーチャー … Web取得 / LLM質問 / メモリ など
           │      📐 アナリスト   … 計算 / 日付 / 単位変換 / JSON / 図表作成 など
           │ 4. ✍️ ライター     … 結果を統合して最終回答を作成
           │ 5. ✅ レビュアー   … 全ステップの成否を検査して報告
           ▼
        最終回答（チャット、図表はSVGで併載）+ 実行トレース（インスペクタ）
```

エージェントを無効化すると、そのエージェント宛だったステップは担当候補の
別エージェントへ自動で再割り当てされる（候補が全滅ならスキップして理由を表示）。

| 層 | 実体 | 役割 |
|----|------|------|
| GUI | `index.html` / `style.css` / `app.js` | チャット・レジストリ・実行タイムライン・インスペクタ |
| エージェントエンジン | `app.js`（ブラウザ内） | 計画 → 実行 → 統合 → レビュー のオーケストレーション |
| サーバー | `server.py`（標準ライブラリのみ） | UI 配信 + `/api/fetch`（SSRF対策つきWeb取得）+ `/api/llm`（LLMプロキシ） |

## 組み込みツールモジュール

| ツール | 担当候補（得意順） | 内容 |
|--------|--------------------|------|
| 🧮 計算機 | アナリスト / リサーチャー | 四則演算・%・べき乗を安全に評価（数字と演算子のみ許可） |
| 📅 日付・時刻 | アナリスト / リサーチャー | ◯日後/前、曜日、日付間の日数 |
| ⚖️ 単位変換 | アナリスト / リサーチャー | 長さ・重さ・温度・データ量・速度 |
| 🔤 テキスト統計 | アナリスト / ライター | 文字数・行数・語数・頻出語 |
| 🧩 JSON 整形 | アナリスト / リサーチャー | 検証 + 整形表示 |
| 🎲 ランダム | アナリスト / リサーチャー | サイコロ・乱数・UUID・ランダム選択 |
| 📊 図表作成 | アナリスト / ライター | 「1月 120、2月 150 を棒グラフに」等から棒・折れ線・円グラフを SVG 生成。直前の JSON 整形結果もデータにできる |
| 🌐 Web 取得 | リサーチャー / アナリスト | サーバー経由でページ取得・本文抽出（プライベートIP拒否） |
| 🗂️ メモリ | リサーチャー / ライター | 「覚えて」で保存 /「思い出して」で検索（localStorage 永続化） |
| ✨ LLM 質問 | リサーチャー / ライター | 設定した LLM への質問・生成（任意） |
| 📝 要約 | ライター / リサーチャー | LLM 接続時は LLM、未接続時は抽出型要約 |

- 各ツール・各エージェントは左パネルのトグルで**有効/無効を切替**できる
- 「＋ 追加」から **カスタムツール**（JavaScript 関数 + 起動キーワード）を登録できる
  （このブラウザ内でのみ保存・実行。信頼できるコードだけを登録すること）
- ツールカードの ▶ で**単体実行**して動作確認できる

## 使用例

- `1280 × (12 + 8) ÷ 4 を計算して、結果をメモに保存して` → 計算機 → メモリ の2ステップ
- `https://example.com/ を取得して要約して` → Web取得 → 要約
- `今日から100日後は何曜日？` / `42.195km をマイルに変換して`
- `1月 120、2月 150、3月 300、4月 210 を棒グラフにして`（折れ線・円グラフも）
- `覚えて: 定例会議は毎週金曜 15:00` → `メモを一覧して`

## LLM 設定

⚙️ 設定 → プロバイダを選択。設定は `kaleido.config.json` にサーバー側保存
（API キーを含むため **git 管理対象外**。`.gitignore` 済み）。

| プロバイダ | ベースURL 既定値 | 備考 |
|-----------|------------------|------|
| ローカルLLM (`local`) | `http://localhost:11434/v1`（Ollama） | OpenAI 互換。LM Studio は `http://localhost:1234/v1`、llama.cpp / vLLM も可。**APIキー任意・プロキシ非経由で直結** |
| OpenAI 互換クラウド (`openai`) | `https://api.openai.com/v1` | API キー必須 |
| Anthropic (`anthropic`) | `https://api.anthropic.com` | API キー必須 |

### プロキシ（社内ネットワーク向け・llmlab と同じ3モード）

設定画面の「プロキシ設定」から指定。**クラウドAIと Web 取得にのみ適用**され、
ローカルLLM（内部アドレス）へは常にプロキシを経由せず直結する。

| 設定 | 動作 |
|------|------|
| プロキシを使う = オフ | 直結（環境変数のプロキシも無視） |
| オン + プロキシURL 空欄 | 環境変数 `HTTP(S)_PROXY` を使用（既定） |
| オン + プロキシURL 指定 | その URL を使用 |

TLS を傍受する社内プロキシの環境では、CA 証明書のパスを
`ca_bundle`（または環境変数 `SSL_CERT_FILE`）で指定できる。検証は常に有効。

### 推論系ローカルLLM

DeepSeek-R1 / QwQ などが本文に混ぜて出力する `<think>…</think>`（および
`reasoning_content` フィールド）は本文から自動分離し、最終回答のみを利用する。

## セキュリティ上の注意

- サーバーは 127.0.0.1 のみ bind。`/api/fetch` は http/https 限定で、
  プライベート/ループバック等へのアクセスとリダイレクトの飛び先を検査（SSRF対策）
- フィード/ページ本文はプレーンテキスト化して返し、UI は全出力をエスケープ後に
  限定的な Markdown として描画（XSS対策）
- カスタムツールは自分のブラウザ内で `Function` として実行される。
  他人のコードを貼り付けないこと
